01 — About

Hands-on, defense-minded, always building

I'm a cybersecurity professional based in North Georgia with a hands-on, build-it-to-understand-it approach to security. My path started in IT support and grew into a deep interest in detection engineering, network security, and security operations.

Rather than learning from slides, I run a production-grade homelab: a virtualized cluster of Linux and Windows systems where I stand up the same tooling used in real SOCs — SIEM/XDR, firewalls, identity services, and threat intelligence — then instrument, attack, and defend them. When the tools I wanted didn't exist, I wrote my own: threat-dash, an open-source security console.

I care about the fundamentals — Active Directory and Group Policy, network segmentation, log analysis, and mapping activity to MITRE ATT&CK — and about turning messy telemetry into something a human can actually act on.

Selected work

threat-dash

// self-hosted security console for the homelab era

A single-container security dashboard that shows who's attacking you, where your containers are reaching out, and lets you shut bad actors down at the edge — all in one pane of glass. Built from scratch with the Python standard library and just two dependencies.

  • Live 3D threat map (globe.gl) plotting attack sources in real time
  • Cloudflare analytics: traffic, WAF blocks, top countries & offending IPs
  • pfSense + Wazuh integrations for firewall and endpoint/SIEM visibility
  • Outbound intel & IOC enrichment — see where workloads phone home
  • One-click edge blocking; pluggable backend client architecture

Lab build

Active Directory & Windows Services Lab

// domain controller, identity, and file services from the ground up

A fully functional Windows domain environment built to internalize enterprise identity and access management — designed, deployed, and validated end-to-end.

  • Windows Server 2019 domain controller with 150+ users and a tiered admin model
  • OU design separating admins, users, and resources; role-based security groups with nesting
  • NTFS permissions driven by group membership; GPO mapped drives via item-level targeting
  • DHCP scope (172.16.0.0/24) with reservations and scope options for gateway/DNS/domain
  • Verified the full chain: authenticate → GPO processing → drive map → scoped file access
